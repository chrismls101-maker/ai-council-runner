**Status:** Locked scope (2026-06-22). Build in phase order.  
**Goal:** Credible daily-driver coding environment — ambient agent + quality gate, not Cursor Tab clone.

**Explicitly out of scope:** Wingman changes (may be removed). Full project LSP (B5b). Competing with Cursor on editor muscle memory alone.

## GLASS_CONTEXT.md — naming (critical)

Two different files share a name. **Do not conflate them.**

| File | Location | Owner | Purpose |
|------|----------|-------|---------|
| **Product doc** | `desktop-glass/GLASS_CONTEXT.md` in *this repo* | Glass team | Internal architecture notes for building Glass |
| **Project memory** | `{userProjectRoot}/GLASS_CONTEXT.md` | **The user's project** | Generated by Code Analyst / “Generate Project Memory”; read by Coder on every run |

**Rules:**
- Project memory is **always** written to the user's `agentCodeWorkspaceRoot`, never into the Glass app bundle or IIVO Research output folder.
- UI copy must say **“saved to your project folder”**, not “Glass context file.”
- Code Analyst `write_file` for memory uses `outputDir: projectRoot` — not agent markdown output path.
- Bootstrap label: `[GLASS_CONTEXT.md — Your project memory file in this repo. Not part of the Glass app.]`

## Phase 1 — Approval fatigue (#1) + git bootstrap (#3)

### 1a. Apply All / Skip All / Trust this run
- **Apply All** — approve current pending edit; auto-approve subsequent `edit_file` / `create_file` for this `runId`.
- **Skip All** — skip current; auto-skip subsequent edits for this `runId`.
- **Trust this run** — same as Apply All (alias in UI).
- **Never** auto-approve `delete_file` — always show approval UI.

**IPC:** `agent-set-approval-mode` `{ runId, mode: "normal" | "trust_edits" | "skip_all" }`  
**Main:** `coderApprovalModeByRunId` map; `createApprovalGate` checks before pushing `agentPendingApproval`.

### 1b. Per-file grouping (UI)
- Inline approval bar shows file path + description once.
- Stream feed: “Waiting for approval — `src/foo.ts`” (existing).

### 1c. Git-aware Coder (no Wingman)
**New modules only** — do not modify `shared/gitDiff.ts` or Wingman panels.

- `shared/coderGitContext.ts` — format `git status --porcelain` + `git diff --stat` for bootstrap text.
- `main/coderGitSnapshot.ts` — run git in `projectRoot` via `execFile`.
- Inject into `buildCoderBootstrapContext` when repo detected.

## Phase 2 — Multi-file tasks (#2) + index defaults (#4)

### 2a. Plan-and-apply UX
- After run completes with N applied files, stream shows **“N files changed”** summary card.
- Coder system prompt nudge: batch related edits; one description per file.
- Optional: queue multiple pending approvals (future) — Phase 1 trust mode unblocks multi-file today.

### 2b. Index reliable by default
- **Auto-index** when user picks workspace (`agentPickWorkspaceRoot`) if `indexEnabled` and no index yet.
- **IDE chip:** `Index · ready (N files)` | `Indexing…` | `No index` + “Index now” link.
- Code Analyst + Coder both use same `searchIndex` path (shipped).

---

## Phase 3 — QA polish (#5) + session continuity (#6)

### 5. QA loop
- Setting: `qaSpeakProgress` (default on) — pipeline TTS cues.
- Stream line on pipeline complete: **“QA · all pass”** or **“QA · N issues”** with link to board.
- **Fix all** button polish — disabled when loop cap hit; clear error copy.

### 6. Session continuity
- Persist `lastCoderSession: { runId, prompt, at }` in settings.
- IDE composer: **“Continue last task”** when session &lt; 24h old.
- **Resume** re-opens IDE with same prompt prefilled (new runId).

### 6b. Project memory UI
- Settings + IDE: clarify project `GLASS_CONTEXT.md` is **in the user's repo**, generated once, optional regenerate.

---

## Phase 4 — Ghost text (#8)

- Setting: `coderGhostTextEnabled` (default off).
- After 800ms idle on a line, request short completion (Haiku) for **current line only**.
- Render as grey Monaco `after` decoration; **Tab** accepts into editor (user edit, not agent write).
- No full-file Tab — line-level only to avoid fighting Coder agent.

**IPC:** `glass-ide-ghost-suggest` `{ relativePath, line, linePrefix }` → `{ suggestion }`

## Phase 4b — Light project intelligence (#7)

- Register up to 120 project TS/JS files as Monaco `addExtraLib` after tsconfig load.
- Cross-file go-to-definition and diagnostics without full LSP server.

**IPC:** `glass-ide-project-libs` → `{ libs: [{ filePath, uri, content }] }`

---

## Phase 5 — Controlled shell (#9)

- New Coder tool: `run_project_command` — allowlist only:
  - `npm run typecheck`, `npm run build`, `npm test`, `npm run lint`
  - `git status`, `git diff`, `npx tsc --noEmit`
- Output returned in tool result (truncated 8KB).
- Terminal errors from IDE PTY can auto-append to next agent turn (existing terminal context path).

---

## Phase 6 — Composer-class orchestration (#10)

### 10a. Analyst → Coder handoff
- When Code Analyst run completes, Response Panel CTA: **“Implement fixes in Glass Coder”**.
- Opens IDE with pre-filled prompt from analysis summary.

### 10b. Checkpoints
- Before each loop-fix iteration, snapshot change log + git diff stat.
- **Rollback iteration** restores from backup paths in change log.

### 10c. Background stream
- Coder run continues with IDE minimizable; Aletheia narrates progress (companion on).
