## What's Actually Happening Under the Surface

### 1. They're all working across multiple tools simultaneously
No-code person: Make, Airtable, browser, ChatGPT tab.
Developer: editor, terminal, browser, Claude, Figma.
Content creator: script, video editor, AI image tool, scheduling platform.
Multi-tool reality is universal.

### 2. They're all moving faster than they can think
The builder era has compressed timelines so dramatically that cognitive overhead is now the bottleneck. Not skill, not tools — the cost of holding context while switching between everything.

### 3. Most don't know what they need until they're stuck
New person doesn't know the right prompt. Experienced person doesn't know the right architecture. Freelancer doesn't know if output is good enough. The gap is not access to AI — it's knowing what to do with it in the moment.

### 4. The skill gap is enormous and widening
Top of the list: building full products in a weekend.
Bottom: still figuring out how to write a prompt.
Glass lives naturally at the top but could pull the bottom up dramatically.

### 5. Almost none of them want another chat interface
They don't want to go somewhere. They want help where they already are.

## The Shared Friction Points

- **API key problem** — create it once, it shows once, you lose it, generate another, now 4 keys and you don't know which is active
- **Environment variables** — .env files scattered across projects, can't remember what you called things
- **Prompt libraries** — perfect prompt from 3 weeks ago is somewhere in a ChatGPT conversation you'll never find
- **Project context switching** — come back after 2 days, spend 20 minutes remembering where you were and what was broken
- **Documentation tabs** — 6 browser tabs open for docs you keep referring back to
- **Credential sprawl** — Supabase key, Stripe key, OpenAI key, Vercel token, GitHub token — all in different places
- **Error patterns** — same class of error shows up across projects, solved from scratch every time

## Product Identity

> Glass is the layer that handles the overhead so builders can stay in flow.

Glass sits above all of it. It already sees the screen. It already knows the project. It's the layer that holds all the scaffolding — surfacing what you need, when you need it, where you already are — without you having to go find it.

The market isn't asking for a better ChatGPT. It's asking for something that stays with them. That's the gap Glass fills — and right now nobody else is in it at the OS level.
